Working with Apache OpenOffice Calc Add-Ins

See Also 

Create and configure a project
  1. Make sure that you have configured a valid Apache OpenOffice and Apache OpenOffice SDK installation under Tools > Options > Miscellaneous > OOo API Plugin.
  2. Use the Apache OpenOffice Calc Add-In project template.
    Choose File > New Project > Apache OpenOffice > Apache OpenOffice Calc Add-In.
  3. Specify the project name.
  4. Specify the Add-In name which is used to create a class name, UNO Idl service and interface name implicitly.
  5. Specify a Java package (implicitly mapped to modules in UNO Idl). Note: This is an optional step.
  6. Select the project location. The project location is the directory where the new project should be created.
  7. Define one or more new Add-In functions and finish the wizard to generate all necessary files and the intial code skeleton for the new Add-In component.
  8. Configure the classpath for the project. You can add a JAR file, a library, or an IDE project to the classpath.


Build the project

  1. Choose Build > Build Main Project or right-click any project and choose Build Project.
  2. With the context menu of the project node (projects window) you can create and deploy an office extension package directly in your specified office installation.
    Right click on the project node > Create OXT
    to create the extension package.
    Right click on the project node > Deploy and Run Extension in Apache OpenOffice
    to deploy the extension and start Apache OpenOffice.
  3. Alternatively, you can chose the appropriate targets in the context menu for the build.xml file.
    Choose build.xml > Run Target > uno-package
    to create the extension package.
    Choose build.xml > Run Target > uno-deploy
    to deploy the extension.

Run the project
  1. In the Project window, open the context menu and select "Deploy and Run Extension in Apache OpenOffice".

Debug the project
  1. Debugging Apache OpenOffice extensions is quite simple. In the Projects window, open the context menu and select "Debug Extension in Target Office".
  2. Apache OpenOffice is started with a new user installation that is created in the build/soffice_debug directory. Your extension is deployed in this new user installation and Apache OpenOffice is started.
  3. Set breakpoints or watches in your code.
  4. Use your Add-In in Apache OpenOffice so that your code is executed.
  5. The code execution will stop at your breakpoint.
