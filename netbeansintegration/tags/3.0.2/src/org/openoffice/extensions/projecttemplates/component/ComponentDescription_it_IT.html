Crea un progetto di tipo Apache OpenOffice UNO component.
Un componente UNO è costituito generalmente dall'implementazione di almeno un servizio specificato nell'UNO IDL. È possibile implementare un servizio esistente di Apache OpenOffice e interfacce addizionali, o definirne di nuovi.
Una procedura guidata consente l'inserimento di informazioni relative alle tipologie implementate, crea un nuovo progetto esteso J2SE di libreria di classi, generando i relativi file UNO IDL (se necessario) e un nuovo file schema sorgente in Java per l'implementazione di tutti i metodi generati dalle tipologie IDL.
