Apache OpenOffice UNO Client Applications

See Also 

An Apache OpenOffice UNO client application is a Java application that provides access to the Apache OpenOffice API. It uses Apache OpenOffice functionality in its own applications via a remote connection. Typical use cases are office automation tasks like:

  • Creating new document based on a template.
  • Extracting data from spreadsheet documents, and working with this data in the client application.

This simple UNO client application requires a local Apache OpenOffice installation on the computer where the application is running because it communicates over a named pipe connection with the Apache OpenOffice process. The application automatically starts a new Apache OpenOffice process, or if an Apache OpenOffice process is already running it connects to this process. If no further option is specified, the default Apache OpenOffice installation is used.

If you prefer to use a specific Apache OpenOffice installation, you can overload the automatic search algorithm by specifying an office installation for the system property com.sun.star.lib.loader.unopath.
For example:

java -Dcom.sun.star.lib.loader.unopath="/opt/staroffice8/program" -jar MyApplication.jar

UNO client applications do not require a special environment to run. The necessary environment is prepared automatically by using a classloader mechanism that knows all the necessary API jar files in the Apache OpenOffice installation.
The application includes some precompiled class files and a Microsoft Windows specific dll from the Apache OpenOffice SDK in the application jar file, and the manifest is extended to make use of a special loader mechanism. The dll is included to make the client application platform independent.

Mainfest entries:

Main-Class: com.sun.star.lib.loader.Loader

Name: com/sun/star/lib/loader/Loader.class
Application-Class: TestMinimalComponent

Bundled class files and dll:

com/sun/star/lib/loader/Loader$CustomURLClassLoader.class
com/sun/star/lib/loader/Loader.class
com/sun/star/lib/loader/InstallationFinder$StreamGobbler.class
com/sun/star/lib/loader/InstallationFinder.class
com/sun/star/lib/loader/WinRegKey.class
com/sun/star/lib/loader/WinRegKeyException.class
win/unowinreg.dll

Additional jar files that may be added standalone or as part of a library are copied into the dist/lib folder of the project when the project is built.
Excluded here are the Apache OpenOffice jars that come with the Apache OpenOffice library. As explained earlier, these jars are alwayes taken from the local Apache OpenOffice installation.
