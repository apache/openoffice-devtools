This panel shows non-localized data that is important for the extension. Note that all the information gathered here will be stored in the description.xml file.

Unique Identifier The Extension is identified by this. As a suggestion, the main package name and the name of the Extension are suggested. Do not use simple names to prevent name clashes in Apache OpenOffice.
Version The version number of the Extension. Suggested is a numbering system as <major>.<minor;>.<micro;>. The numbering is started with a default of 0.0.1.
Dependency Version It may be that the Extension needs a particular OpenOffie.org version to work correctly, e.g. when a feature is used that was implemented for that version. Possible value is "3.1" meaning of course OpenOffie.org 3.1
Dependency Name The dependency name is always OpenOffie.org followed by the version number. Therefore it is only displayed here without editing possibility.
Update URLs One or more update URLs can be given to notify the user of an Extension, that a new version of the Extension is available.
